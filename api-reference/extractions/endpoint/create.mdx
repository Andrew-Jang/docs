---
title: "Create Extraction Schema"
openapi: "openapi-extractions POST /v1/extractions"
---

The **Create Extraction Schema** endpoint allows you to define a new extraction schema for a specific document type in the ParDocs API. An extraction schema specifies the rules and structure for extracting data from documents. By creating an extraction schema, you can customize how data is extracted based on your specific needs, which can then be applied to various documents for processing.

<Info>Note that `{extraction_type}` refers to the type of document.</Info>

**Request Body Parameters:**

- `extraction_type` (string): Specifies the type of document the schema applies to.
- `extraction_data` (object): Contains the detailed schema definition for the extraction.

<RequestExample>
```bash cURL
curl --request POST \
  --url https://api.pardocs.com/v1/extractions \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: <api-key>' \
  --data '{
  "extraction_type": "<string>",
  "extraction_data": {}
}'
```
```python Python
import json
import requests

url = "https://api.pardocs.com/v1/extractions"
headers = {
    "x-api-key": "<api-key>",
    "Content-Type": "application/json"
}
data = {
    "extraction_type": "<string>",
    "extraction_data": {}
}

response = requests.post(url, headers=headers, data=json.dumps(data))

print(response.json())
```
</RequestExample>
